**Changes Made:**
1. **FEATURE: VIX-Adaptive Exit Thresholds** — `_manage_exits()` now reads `last_vix` from daily state and adjusts profit target, stop loss, and trailing stop thresholds based on VIX regime:
   - Low VIX (<15): 40% profit / 25% stop / 25% trail activate / 12% trail stop
   - Normal VIX (15-20): 50% / 30% / 30% / 15% (matches current static defaults)
   - High VIX (20-30): 60% / 40% / 35% / 18% (wider for current regime)
   - Crisis VIX (>30): 70% / 50% / 40% / 22%
   Falls back to static config values when VIX data unavailable. VIX is persisted to daily state during market context gathering (Phase 2), used by exit management (Phase 1) in subsequent cycles.
   **Rationale:** With VIX at 24.54 (HIGH), the static 50% profit / 30% stop is too tight. Options premiums swing more in high-vol environments — a 30% stop gets hit by noise before the directional thesis plays out. The 60/40 thresholds give trades room to breathe while still controlling risk. This was identified as a priority in the April 2 12:00 ET and 14:00 ET reports.

2. **IMPROVEMENT: LLM System Prompt — Selectivity Bias** — Changed all three LLM clients (Ollama, Anthropic, OpenAI) from "make a trading decision" to "Your DEFAULT decision is NO_TRADE. Only output TRADE when confidence is above threshold AND direction is clear AND risk/reward is favorable."
   **Rationale:** The original prompt ("analyze and make a trading decision") implicitly biases 8B models toward outputting TRADE — the word "decision" implies there's always something to decide. The new prompt explicitly establishes NO_TRADE as the baseline, which aligns with the strategy principle that most cycles should not trade. This is especially important for Plutus (8B) which tends to be more compliant with prompt framing.
