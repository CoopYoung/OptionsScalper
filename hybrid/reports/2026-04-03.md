### CODE REVIEW NOTES

**Bug found and fixed:**
- `is_market_hours()` in `risk/validator.py` did not check for market holidays. On Good Friday (today), the cron job would execute, `is_market_hours()` would return True (Friday = weekday 4), and the bot would attempt to call Alpaca APIs on a closed market. This would either error out (wasting cycles) or worse, act on stale data. **Fixed by adding a 2026 NYSE holiday calendar set.**

**Architectural observations (no changes — need trade data first):**
1. The `_extract_json` regex in `llm.py:37` (`[^{}]*`) won't match nested JSON with inner braces. Tries 1 and 3 handle this correctly, so it's a non-issue in practice, but could be cleaned up.
2. `config.py` doesn't have a FINNHUB_API_KEY variable exposed — it's loaded via dotenv but not referenced. The digest.py or CLI modules likely read it directly from env. This is fine but inconsistent with the pattern.
3. The `validate_new_order` function imports `SIGNAL_CONFIDENCE_THRESHOLD` inside the function body (line 88-89 of validator.py) rather than at the top. This is a minor style issue.
4. No code changes warranted beyond the holiday fix — zero trade data means zero statistical basis for parameter tuning.
