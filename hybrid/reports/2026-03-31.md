## Code Review & Analysis

The codebase is well-structured and production-ready for paper trading. The modular architecture (broker, digest, LLM, risk, alerts) is clean. However, several issues prevent the bot from ever executing trades:

**Root Cause Analysis:**

2. **No live data flowing** — Without cron running during 9:45-15:00 ET on weekdays, the bot has never seen real market conditions.

### Issues Found in Code Review

#### Issue 1: `_find_todays_expiry()` skips Tuesday/Thursday incorrectly
In `digest.py:714`, the function has this logic:
```python
if symbol in ("SPY", "QQQ") and weekday not in (0, 2, 4):
    pass  # Tue/Thu — these tickers usually don't have 0DTE
```
This is correct behavior — it still tries the API call. But it wastes an API call on days that will never have 0DTE. Minor optimization: return None early on Tue/Thu for SPY/QQQ.

#### Issue 2: HARD_CLOSE_ET is 15:45, should be 15:15
In `config.py:71`, `HARD_CLOSE_ET` defaults to `"15:45"`, but CLAUDE.md says the bot should hard-close at 15:15 ET (Alpaca auto-liquidates at 15:30). The 15:45 setting risks Alpaca force-liquidating positions at worse prices before the bot closes them.

#### Issue 3: ENTRY_CUTOFF_ET is 15:00, CLAUDE.md says 14:30
In `config.py:70`, `ENTRY_CUTOFF_ET` defaults to `"15:00"` but CLAUDE.md specifies `ENTRY_CUTOFF=14:30`. Entering 0DTE positions after 14:30 has extreme theta decay risk.

#### Issue 4: Buying power check uses float comparison
In `validator.py:134-141`, `buying_power` is parsed from `account.get("buying_power", 0)` which may return a string from the API. The code should ensure it's a float.

#### Issue 5: Ollama output parsing may miss nested JSON
In `llm.py:37`, the regex `\{[^{}]*"decision"[^{}]*\}` won't match nested objects (e.g., if the model wraps the response in another object). This is fine for the simple schema requested, but worth noting.

#### Issue 6: No system prompt for Ollama
The `OllamaClient._call()` method sends only a `prompt` field. There's no system prompt telling the model its role. The Plutus model may benefit from a system prompt like "You are a quantitative options trading analyst. Respond only with JSON."
